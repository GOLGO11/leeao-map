# 数据方法

## 目标

把李敖一生可定位的空间轨迹整理成地图数据。核心对象不是“地点清单”，而是“某个时间段里，李敖在某地发生了什么，并由哪段文本支持”。

## 工作流

1. 选定一本书或一组章节。
2. 建立来源记录，记录书名、文件路径、范围、处理日期。
3. 初筛一轮：快速通读和检索，抽取候选地点事件，包括时间、地点、动作、人物、作品、机构。
4. 复查一轮：回到原文逐条核对，确认是否为李敖本人足迹、是否足够重要、是否有明确证据。
5. 分级判断：
   - `high`：原文直接给出时间、地点和事件。
   - `medium`：原文直接给出地点事件，但时间或地点精度较粗。
   - `low`：需要跨文本推断，暂不进入主线或标为待复核。
6. 拆分地点和事件：
   - `places.geojson` 只保存地点实体和坐标。
   - `events.json` 保存时间、叙事、来源和地点关联。
7. 复核坐标：
   - 门牌、旧地名、机构旧址必须标注坐标精度。
   - 无法确认门牌时，使用城市级或区级坐标，并标记 `needs_review`。
8. 进入地图展示或动画路线。

每本书至少两轮：第一轮初筛，第二轮复查。初筛可以先把重要候选点放入笔记或临时清单；复查后才把主线事件正式写入 `events.json`。如果某本书信息密度高，复查可以继续拆成第三轮、第四轮。

## 文本编码

本地全集中的部分 `.txt` 文件不是 UTF-8，而是中文 Windows 环境下常见编码。人工阅读时 PowerShell 默认读取通常可显示中文；程序化检索时需要显式按 `gb18030` 解码，否则会出现乱码或漏检。第二轮《李敖自传与回忆》即按 `gb18030` 解码检索。

## 数据字段约定

### 地点

- `id`：稳定地点 ID。
- `name_zh`：中文名称。
- `aliases`：旧名、别名、写法差异。
- `place_type`：`city`、`address`、`institution`、`region`、`site` 等。
- `coordinate_precision`：`exact`、`institution_approx`、`address_approx`、`city`、`region`、`unknown`。
- `geometry`：GeoJSON Point，坐标顺序为 `[lng, lat]`。
- `needs_review`：坐标或地点解释是否需要人工复核。

### 事件

- `id`：稳定事件 ID。
- `sequence`：主线播放顺序。
- `date_start` / `date_end`：ISO 日期；只有年份时使用 `YYYY-01-01` 到 `YYYY-12-31`。
- `date_label`：原文时间表达。
- `title`：地图卡片标题。
- `summary`：简短说明。
- `place_ids`：关联地点。
- `category`：出生、迁徙、求学、居住、服役、关押、出版、演讲、晚年等。
- `certainty`：对时间地点事件三者绑定的把握。
- `source_refs`：来源文件、来源定位、短摘。

## 第一批地图策略

第一轮只做“人生主干路线”，优先展示：

哈尔滨出生 -> 北平/北京童年求学 -> 天津/上海转渡 -> 台中落脚 -> 台北台大 -> 凤山/澎湖服役 -> 台北与新店居住写作 -> 军法处与土城关押 -> 台北电视与公共活动 -> 2005 大陆演讲 -> 晚年台北。

后续每本书会补充：

- 具体门牌与旧址考证。
- 同一地点的多次事件。
- 关联原文、照片、视频、作品。
- 对“地名只是谈论对象、不是李敖足迹”的排除记录。
